WHAT EACH PART IS FOR

- `config.py`
  Loads all environment variables defined in codex_identity.md, including
  `CLIP_PROMPT_VERSION`, upload limits, chunk settings, and polling interval.

- `requirements.txt`
  Must contain the exact pinned dependencies listed in codex_identity.md.

- `docker-compose.yml`
  Starts local Redis and Postgres for Celery and the jobs table during development.

- `api/main.py`
  FastAPI entry point. Registers routes and exposes the backend app.

- `api/routes/upload.py`
  Implements `POST /upload`.
  Validates format, file size, and duration before creating a job.

- `api/routes/jobs.py`
  Implements:
  `GET /jobs/{job_id}/status`
  `GET /jobs/{job_id}/clips`

- `api/models/job.py`
  Canonical Pydantic models only.
  All routes, workers, and services must import from this file.

- `workers/celery_app.py`
  Celery configuration and Redis connection.

- `workers/pipeline.py`
  Full background job pipeline:
  extract audio -> transcribe -> detect clips -> cut -> crop -> caption -> export.

- `services/storage.py`
  Supabase Storage upload and download helpers for source video, audio, and final clips.

- `services/transcription.py`
  Whisper wrapper with word-level timestamps and retry logic via `tenacity`.

- `services/clip_detector.py`
  Transcript chunking, prompt loading, LLM scoring, overlap removal, thresholding,
  and final clip selection.

- `services/video_processor.py`
  FFmpeg audio extraction, raw clip cutting, vertical crop, and final export logic
  using the exact commands defined in codex_identity.md.

- `services/caption_renderer.py`
  Converts Whisper word timestamps into clip-relative SRT captions and burns them in.

- `services/cost_tracker.py`
  Calculates estimated cost before processing and accumulates actual API cost after calls.

- `db/connection.py`
  SQLAlchemy database engine setup.

- `db/queries.py`
  All database queries for the `jobs` table.
  No inline SQL should appear in routes, workers, or services.

- `migrations/001_create_jobs_table.sql`
  Must contain the exact SQL from codex_identity.md, including indexes and
  the `updated_at` trigger.

- `prompts/clip_detection_v1.txt`
  The active default clip-detection prompt file.
  Prompt versions should stay in this folder as `clip_detection_v2.txt`, etc.

- `web/`
  Minimal Next.js frontend required by the MVP:
  upload one video, poll job status every 4 seconds, preview clips, and download them.

- `tests/`
  Verifies the strict contract defined in codex_identity.md:
  file validation, job status responses, clip scoring rules, FFmpeg steps, and captions.
